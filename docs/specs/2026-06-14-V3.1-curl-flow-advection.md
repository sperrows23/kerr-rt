# V3.1 — Curl-flow advection (animate the CKS-18 warp)

**Status:** SCOPE — *not yet owner-approved, not yet a SKILL.md formula.* This document
frames the design and surfaces the owner decisions that must be made (one-variable-at-a-
time) **before** any code is written. V3.0 (static warp, CKS-18) is shipped + GPU-verified
(commit `c63639b`); V3.1 is the second half of the owner's V3 staging decision
("static warp first → curl-flow advection", the D2.2→D2.3 split).

**Math of record (proposed):** an extension of **Formula CKS-18**, same VISUALIZATION
governance class as CKS-12 §2/§3. The proposed math below is a *candidate* — it must be
ratified into SKILL.md (as CKS-18 §2, or a new CKS-19) and owner-approved before
implementation, exactly as CKS-18 itself was approved before V3.0.

## Problem

V3.0 makes the eddies **static in the gas's material frame**: the divergence-free warp is
frozen (fixed `curl.seed`), and only the CKS-12 §2 Keplerian shear winds it azimuthally
into filaments. Real turbulence is not frozen — eddies *form, stretch, merge, and
dissipate*; the structure boils. V3.1 makes the curl potential `ψ` **time-dependent** so
the eddies evolve over `t_disk`, while still being wound by §2. The two motions compose:
§2 is the laminar bulk orbit (winding), V3.1 is the in-place turbulent evolution (boiling).

## What stays fixed from V3.0 (owner decisions to re-confirm, not re-open)

1. **In-plane `(u,φ)` only.** ζ (vertical) churn remains deferred to the later "3D curl"
   increment (vector potential). V3.1 keeps the V3.0 scope: ζ untouched.
2. **Texturing, not transport.** V3.1 remains a time-evolving divergence-free
   *displacement of the noise sample coordinate* — NOT a semi-Lagrangian self-advection of
   a conserved density (continuity solve). A genuine fluid transport solve STOPs for skill
   extension (CLAUDE.md policy / CKS-18 governance §). "Curl-flow advection" here means
   *the texture coordinate is displaced by a time-varying incompressible field*, the
   Neyret advected-texture sense — the same sense CKS-12 §2 already uses.
3. **Never touches GR.** `p_μ`, `u^μ` (CKS-8), `g` (CKS-9), the `g⁴` exponent, the `f_PT`
   shape (CKS-11), the blackbody chroma form — all untouched. Amplitude/coordinate only.

## Core design decision — how to make ψ time-dependent

Two candidate mechanisms. **This is the pivotal owner decision for V3.1.**

### Option A (recommended) — reuse the §2 dual-phase reset blend

Mirror CKS-12 §2 *exactly*, applied to the curl potential instead of the density domain.
The potential gets a per-cycle reseed and a triangle-weighted crossfade between two
staggered phases, so the animation is **bounded, seamless at reseeds (C0-continuous), and
non-repeating** — all three already proven and tested for §2.

```
s_c   = t_disk / T_c                      # T_c = curl flow period (see clock decision)
α_k   = fract(s_c + k/2),  k ∈ {0,1}      # phase age
γ_k   = floor(s_c + k/2)                  # phase cycle index (→ reseed)
ω_k   = 1 − |2·α_k − 1|                   # triangle weights, ω_0 + ω_1 ≡ 1

ψ_k(u,φ) = sfbm3( P(u,φ) ; hash(curl.seed, k, γ_k) )      # P = the CKS-18 cylinder embedding
ψ(u,φ;t) = ω_0·ψ_0 + ω_1·ψ_1
δu = +∂ψ/∂φ ,  δφ = −∂ψ/∂u                # central FD, exactly as CKS-18
```

Divergence-free is preserved under the blend: curl is linear, so
`curl(ω_0·ψ_0 + ω_1·ψ_1) = ω_0·curl(ψ_0) + ω_1·curl(ψ_1)` — a convex combination of
divergence-free fields is divergence-free. Seamlessness in φ is inherited per-phase from
the cylinder embedding (CKS-18). So all three V3.0 invariants survive unchanged.

- **Pro:** reuses the proven, tested §2 machinery (bounded, C0 at resets, non-repeating);
  no new noise primitive; the "static → animate" diff is small and auditable.
- **Pro:** `T_c ≤ 0` (no curl clock) ⇒ collapses to a single fixed-seed phase = the V3.0
  static warp **bit-for-bit** (the exact mirror of §2's "shear_period ≤ 0 ⇒ static path
  bit-identical"). Free regression safety.
- **Con:** the eddies "reset-blend" rather than evolve by a true flow — a crossfade boil,
  not a continuous drift. In practice indistinguishable for an offline beauty disk, and
  it is precisely how §2 already animates.

### Option B — 4D simplex potential (continuous time axis)

Add a time axis to the embedding and evaluate a **4D** simplex fBm:
`ψ(u,φ;t) = sfbm4( cosφ·ρ_c, sinφ·ρ_c, u·k_u, t·k_t )`. The potential drifts smoothly
through a frozen 4D field — a genuinely continuous boil, no reset crossfade.

- **Pro:** smoothest possible evolution; no reset cadence to tune; conceptually clean.
- **Con:** requires a **new noise primitive** `snoise4`/`sfbm4` + `@ti.func` twin
  (the V1.5 basis only goes to 3D) — a non-trivial library addition with its own
  isotropy/twin-parity guards. Larger surface, more JIT, slower per-eval (4D simplex has 5
  corners vs 4). Heavier than the problem warrants for V3.1.

**Recommendation:** **Option A.** It is the minimal, proven, bit-identity-safe path and is
symmetric with how the disk already animates (§2). Reserve Option B (the 4D basis) for a
future increment if the crossfade boil ever reads as unnatural.

## Composition with §2 (additive, material-frame)

The warp is still applied at the entry of `_disk_noise_m` / `_mod_fbm4`, on the
already-sheared per-phase `φ′_k` (CKS-12 §2) — i.e. the eddies live in the material frame
and §2 winds them, exactly as V3.0. The only change is that the displacement
`(δu, δφ)` now depends on `t_disk`:

```
u'   = u   + A·δu(u, φ′_k; t_disk)
φ′_k = φ′_k + A·δφ(u, φ′_k; t_disk)        # additive on top of the §2 shear
```

This matches the spec's deferred note: *"animate ψ into time-dependent curl-flow advection
… composed additively with the §2 Keplerian shear."*

## Secondary owner decision — the curl flow clock `T_c`

Does the eddy boil run on its **own** clock or the shear clock?

- **B1 (recommended): a new base dial `disk.noise.curl.flow_period_M`** (geometric M),
  decoupled from `dynamics.shear_period_M`. Turbulence eddy-turnover and bulk-orbit
  winding are physically independent timescales, so an independent dial gives the most
  natural look control. Default chosen so V3.1 is OFF by default (see below).
- **B2: derive `T_c` from `dynamics`** (e.g. a multiple of `shear_period_M` via the CKS-13
  resolver). Fewer dials, but couples two unrelated motions and would touch the resolver
  (CKS-13) — V3.0 deliberately avoided a resolver change.

A `curl.flow_dynamism` gain (mirror of `disk.noise.dynamism`) is also proposed — emphasise
the boil rate for a given frame without changing the reset cadence (C0 at resets holds).

## Config — extend `disk.noise.curl` (no resolver change if B1)

```yaml
disk:
  noise:
    curl:
      enabled: false        # unchanged master gate
      amp: 0.0              # unchanged
      # ... existing CKS-18 static dials (freq_phi, freq_u, octaves, lacunarity,
      #     gain, seed, fd_eps) ...
      flow_period_M: 0.0    # NEW — T_c; ≤ 0 ⇒ static (V3.0) warp, bit-identical
      flow_dynamism: 1.0    # NEW — boil-rate gain (1.0 = no emphasis)
```

If **B1** is chosen these are pure base look dials → **no CKS-13 resolver change** (V3.0's
property is preserved). The values flow through `_setup_disk_noise`; the parameter buffer
grows past the current `_NOISE_N = 52`.

## Bit-identity / regression safety

- `curl.enabled: false` (or `amp: 0`) ⇒ identical to V2, unchanged from V3.0.
- `flow_period_M ≤ 0` ⇒ single fixed-seed phase = the **V3.0 static warp bit-for-bit**.
  This is the key regression hook: existing V3.0 / default-off goldens stay valid; the
  curl-on V3.0 look is also preserved exactly when the clock is off.

## Tests (extend the V3.0 suite)

CPU (`tests/test_noise.py`) + GPU twin (`tests/test_noise_gpu.py` / `test_disk_noise.py`):

- **Divergence-free at each t** — FD-verify `∇·(δu, δφ) ≈ 0` on a `(u,φ)` grid at several
  `t_disk` (the blend preserves it; this guards the linearity argument).
- **Seamless at each t** — warped coord continuous across φ=0 ≡ 2π at several `t_disk`.
- **Reset C0-continuity in time** — `ω_k → 0` exactly at each phase reset ⇒ the warp is
  continuous through reseeds (the §2 property, now in the time axis).
- **Static-fallback bit-identity** — `flow_period_M ≤ 0` reproduces the V3.0 static warp
  **bit-for-bit** (mirror of §2's static-fallback test).
- **Determinism** — same seed + same `t_disk` ⇒ identical warp; no `ti.random`.
- **Twin parity** — `curl_warp`/`curl_warp_ti` agree to the **same derived
  `amp·_SATOL/fd_eps` bound** as V3.0 (the warp is still a derivative; the blend does not
  change the FD-amplification analysis).
- **Evolution smoke** — the warped density field changes between distinct `t_disk` and
  stays finite / NaN-free.

## Cost

Option A doubles the per-sample curl evals (two phases × the CKS-18 4-eval central
difference × octaves), for density AND modulation — ~2× V3.0's curl cost, parallel per
sample and offline. Accepted, same trade as §2's dual-phase. (Option B is ~1.25× per eval
but adds 4D simplex.)

3. **V3.0 scope re-confirmed:** in-plane `(u,φ)` only (ζ deferred to the later 3D-curl
   vector-potential increment), and texturing-not-transport (no semi-Lagrangian continuity

## Deferred beyond V3.1 (unchanged from the CKS-18 spec)

- **3D curl** — a vector potential `A(cosφ,sinφ,u,ζ)` whose curl also displaces ζ, so the
  V2 flared bulk churns vertically (~3× the evals).
- **Analytic simplex gradient** — replace the 4-eval central difference (Gustavson simplex
  has a closed form).
- **A dedicated curl-on golden frame** (relational, V5 sign-off).
